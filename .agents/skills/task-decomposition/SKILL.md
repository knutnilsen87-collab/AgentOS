---
name: task-decomposition
description: Use when a task is broad, multi-step, high-risk, or spans planning, execution, review, and documentation updates.
---

Break the task into:
1. objective
2. constraints
3. assumptions
4. required inputs
5. implementation steps
6. verification steps
7. rollback or recovery considerations
8. explicit exit criteria

Prefer smaller, reviewable chunks over one large change.
