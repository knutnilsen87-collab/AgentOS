# Safety and Approval Model

## Principle
Autonomy should grow only when boundaries, verification, and recovery improve.

## Initial operating stance
Default to safe, reviewable work:
- read first
- plan second
- edit in controlled scope
- verify before claiming success

## Recommended approval posture
- use conservative approval settings until the scaffold and test loop exist
- do not use unrestricted machine-wide autonomy for early bootstrap work
- keep project boundaries tight

## Safety requirements for the future product
- sandbox task execution
- controlled network access
- protected paths
- explicit approval checkpoints
- replayable audit logs
- rollback capability
- secrets isolation

## Immediate repo rule
Do not represent speculative implementation as verified behavior.
